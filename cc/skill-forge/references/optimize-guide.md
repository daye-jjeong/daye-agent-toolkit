# 최적화 상세 가이드

## 이동 기준 테이블

| 콘텐츠 유형 | 판단 | 이유 |
|-------------|------|------|
| 핵심 워크플로우 단계 | **유지** | 스킬 실행에 필수 |
| 트리거/모드 설명 | **유지** | 모드 판단에 필수 |
| 출력 포맷 (간략) | **유지** | 결과물 형태 명시 |
| 상세 예시 (3줄+) | **이동** | 필요 시 Read로 참조 |
| 큰 테이블 (5행+) | **이동** | 본문 비대화 방지 |
| 템플릿 모음 | **이동** | 필요 시만 로드 |
| 배경 지식/이론 | **이동** | 실행에 불필요 |
| 도구 사용법 상세 | **이동** | 기본 지식으로 충분 |

## 참조 포인터 패턴

이동한 콘텐츠 자리에 다음 포맷으로 포인터를 남김:

```markdown
**상세 예시**: `{baseDir}/references/examples.md` 참고
```

포인터 작성 규칙:
- `{baseDir}`는 스킬 루트 디렉토리 (예: `skill-forge/`)
- 한 줄로 간결하게, bold로 제목 표시
- references/ 파일명은 kebab-case

## 최적화 우선순위

줄 수가 많은 섹션부터 이동:

1. 코드 블록 (보통 5-15줄 절약)
2. 큰 테이블 (보통 5-10줄 절약)
3. 상세 예시 (보통 3-8줄 절약)
4. 배경 설명 (보통 5-20줄 절약)

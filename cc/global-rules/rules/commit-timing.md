# 커밋 타이밍과 보고 형식

## 커밋 제안 조건

다음 상황에서만 커밋하라:
1. plan 태스크 1개 완료 + 관련 테스트 통과 시 (태스크 경계)
2. simplify/pr-review 루프 수렴 후 (마무리 단계는 수렴 후 한 번만)
3. 세션 종료/핸드오프 시 (WIP 커밋)
4. 사용자가 명시적으로 요청 시

파일 수정 직후, 태스크 진행 중, simplify/pr-review 루프 중간에는 커밋을 묻지 마라.

## 보고 형식

"커밋할까요?"만 단독으로 묻지 마라. 반드시 **현재 위치 + 다음 행동**을 함께 보고하라.

❌ "커밋할까요?"
✅ "태스크 2/4 완료, 테스트 통과. 커밋하고 태스크 3 진행할게."
✅ "구현 + 검증 완료. 커밋하고 simplify → pr-review로 넘어갈게."
✅ "simplify/pr-review 수렴됐다. 최종 커밋하고 완료 보고할게."

Why: 모델의 기본 패턴이 "파일 수정 → 커밋할까요?"로 되어 있어 워크플로우의 검증 단계를 건너뛰고, 사용자가 다음 단계를 직접 판단해야 하는 부담을 준다. 워크플로우를 알고 있으면 현재 위치와 다음 행동을 함께 보고해야 한다.

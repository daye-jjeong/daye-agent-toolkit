# 작업 시작 전 확인

코드를 수정하기 전에 현재 상태를 확인하라.
Why: 확인 누락 → 뒤늦은 인지 → 디버깅 루프. 가장 흔한 실수 유형.

## 확인 항목

1. 관련 파일을 먼저 읽어라. CLI 명령, 시스템 컴포넌트의 역할을 가정하지 말고 소스를 확인
2. 대상 시스템의 현재 상태: 설정, 환경변수, 브랜치, 배포 상태
3. 내 가정 명시: "X가 Y일 것이다"를 말로 적고, 실제 값을 확인
4. 이미 구현된 것을 다시 만들거나, 존재하는 패턴을 무시하지 마라

확신이 없으면 "이렇게 이해했는데 맞나요?"라고 물어라.

# Pre-flight: 작업 시작 전 상태 확인

새 작업을 시작할 때 코드를 건드리기 전에 현재 상태를 확인하라.
Why: 환경/설정을 확인하지 않고 진행하면 "뒤늦게 발견" → 디버깅 루프에 빠진다.
프로파일 데이터에서 mistake 최다 유형이 "확인 누락 후 뒤늦은 인지"였다.

## 확인 항목

1. **대상 시스템의 현재 상태**: 설정, 환경변수, 모델, 브랜치, 배포 상태
2. **내 가정 명시**: "X가 Y일 것이다"를 말로 적고, 실제 값을 확인
3. **이전 실수 확인**: 같은 영역의 correction rule이 있는지 `.claude/rules/correction-*` 확인

## 적용 시점

- 새 피처/버그 작업 시작 시
- 디버깅을 시작하려 할 때 ("왜 안 되지?" 전에 "현재 상태가 뭐지?"부터)
- 다른 레포/환경으로 전환할 때

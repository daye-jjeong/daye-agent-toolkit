# 개발자를 위한 이메일 템플릿

소프트웨어 개발 업무에서 자주 사용하는 이메일 템플릿 모음입니다. 수신자와의 관계 및 상황에 맞게 톤과 내용을 조절하여 사용하세요.

> 본 문서는 [softaworks/agent-toolkit](https://github.com/softaworks/agent-toolkit)의 영문 이메일 템플릿을 한국 비즈니스 이메일 관행에 맞게 현지화한 버전입니다.

---

## 1. 상태 보고 이메일 (Status Update)

```markdown
제목: [프로젝트명] 제N주차 진행 현황 보고

[팀명/이름]님 안녕하세요.
[본인 이름]입니다.

[프로젝트명] 금주 진행 현황을 공유드립니다.

**요약:** [전체 진행 상황과 주요 성과를 한 문장으로 요약]

**금주 완료 사항:**
- [완료한 기능/작업 및 결과 요약]
- [완료한 기능/작업 및 결과 요약]

**진행 중:**
- [진행 중인 작업] - 완료 예정일: [날짜]
- [진행 중인 작업] - 완료 예정일: [날짜]

**블로커/리스크:**
- [블로커 설명] - 필요 사항: [해결을 위해 필요한 것]

**차주 계획:**
- [예정된 작업 항목]

궁금하신 점이나 추가 설명이 필요하신 부분이 있으시면 말씀해 주세요.

감사합니다.
[본인 이름] 드림
```

## 2. 스프린트 회고 요약 (Sprint Retro Summary)

```markdown
제목: 스프린트 [N] 회고 요약

팀원 여러분 안녕하세요.
[본인 이름]입니다.

금일 회고에 참여해 주셔서 감사합니다. 논의된 내용을 아래와 같이 정리해 드립니다.

**잘된 점 (Keep):**
- [긍정적 항목 1]
- [긍정적 항목 2]

**개선이 필요한 점 (Problem):**
- [개선 영역 1]
- [개선 영역 2]

**액션 아이템 (Try):**
- [ ] [담당자]: [액션 내용] - 기한: [날짜]
- [ ] [담당자]: [액션 내용] - 기한: [날짜]

다음 스프린트 회고에서 위 액션 아이템의 진행 상황을 함께 점검하겠습니다.
수고 많으셨습니다.

감사합니다.
[본인 이름] 드림
```

## 한국 비즈니스 이메일 작성 팁

### 기본 원칙

1. **존댓말 유지** - 상대방과 친한 사이라도 업무 이메일에서는 존댓말(합니다체)을 사용하세요. 팀 내 합의가 있는 경우에만 반말 또는 해요체를 사용합니다.
2. **자기소개 포함** - 이메일 서두에 "[본인 이름]입니다"를 넣어 발신자를 명확히 밝히세요. 잘 모르는 사이일수록 소속과 이름을 함께 밝힙니다. (예: "[팀명]의 [이름]입니다")
3. **불필요한 섹션 삭제** - 블로커가 없으면 블로커 섹션을 포함하지 마세요. 없는 내용을 억지로 채우지 않습니다.
4. **수신자에 맞는 격식 조절** - 같은 팀 동료에게는 다소 가볍게, 경영진이나 타 부서에는 보다 격식 있게 작성합니다.
5. **교정 확인** - 특히 이름, 직함, 날짜, 숫자를 반드시 재확인하세요. 상대방의 이름이나 직함을 틀리는 것은 큰 실례입니다.

### 한국 비즈니스 이메일 관행

6. **제목에 말머리 활용** - `[공지]`, `[요청]`, `[긴급]`, `[FYI]`, `[회신 요청]` 등의 말머리를 제목 앞에 붙이면 수신자가 이메일의 성격을 빠르게 파악할 수 있습니다.
7. **배려 표현 활용** - "바쁘신 중에 죄송합니다", "번거로우시겠지만" 등 상대방에 대한 배려 표현을 적절히 사용하면 부드러운 인상을 줍니다. 다만 과도하게 사용하면 본론이 묻힐 수 있으니 적절히 조절하세요.
8. **"드림" 사용** - 이메일 끝에 "[이름] 드림"은 격식 있는 마무리 표현입니다. 가까운 동료에게는 "[이름] 올림" 또는 이름만 적어도 무방합니다.
9. **발송 시간 고려** - 퇴근 이후나 주말에 보내야 하는 경우 "퇴근 후 메일 드려 죄송합니다", "주말에 메일 드려 죄송합니다"와 같은 표현을 추가하거나 예약 발송을 활용하세요.
10. **참조(CC) 활용** - 한국 업무 문화에서는 관련 상급자를 CC에 포함하는 경우가 많습니다. 에스컬레이션이나 부서 간 요청 시에는 양측 매니저를 CC에 넣는 것이 일반적입니다.

### 상황별 맺음말 가이드

| 상황 | 추천 맺음말 |
| --- | --- |
| 일반적인 업무 메일 | 감사합니다. |
| 요청/부탁하는 메일 | 잘 부탁드립니다. / 확인 부탁드리겠습니다. |
| 보고 메일 | 감사합니다. / 이상입니다. |
| 사과/양해 구하는 메일 | 양해 부탁드립니다. / 죄송합니다. |
| 축하/격려 메일 | 수고 많으셨습니다. / 고생 많으셨습니다. |
| 첫 인사 메일 | 잘 부탁드리겠습니다. |

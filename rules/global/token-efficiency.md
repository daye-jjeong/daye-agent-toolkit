# 토큰 효율

## grep 전에 자문
"이 정보가 있을 파일을 이미 아는가?" → 알면 Read. 모르면 files_with_matches로 파일 특정 → Read.
broad grep으로 20KB+ 결과 금지.

## 편집은 묶어서
같은 파일 관련 변경 → 한 번에. 파라미터 추가 + docstring + 호출부를 3번 나눠 Edit 금지.

## 문서는 한 곳에
같은 내용 두 파일에 쓰지 마라. 정보 주인 정하고 나머지는 링크/요약만.

## 전체 재작성 지양
변경 30% 미만이면 Write 대신 Edit.

## Explore 에이전트
대상 파일을 알거나 grep 2~3개로 찾을 수 있으면 Explore 쓰지 마라.

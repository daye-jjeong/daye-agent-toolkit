# Progressive Disclosure 패턴 + 컨텐츠 스펙 + 안티패턴

## Progressive Disclosure 패턴

### 패턴 A: 접기 (Collapsible)
```html
<details>
  <summary>센서 이력 (12건)</summary>
  <!-- 테이블 -->
</details>
```
- **사용**: L2 테이블, 이력 목록
- **기본 상태**: 접힘
- **summary에 건수 표시**: 펼치지 않아도 데이터 존재 여부를 알 수 있게

### 패턴 B: 탭 그룹
```
[현재 상태] [이력] [설정]
```
- **사용**: L1-L2가 3개 이상인 사이드 패널
- **첫 번째 탭**: 가장 자주 보는 정보 (보통 "현재 상태")
- **탭 이름**: 2-3글자. "캘리브레이션 샘플 이력" → "캘리"

### 패턴 C: 요약→상세
```
에러 3건 [상세 보기 →]
```
- **사용**: L0에 L1 정보를 암시할 때
- **요약은 숫자 1개 + 상태**: "에러 3건", "대기 5건"
- **상세는 클릭 시 노출**

### 패턴 D: 조건부 노출
```
정상일 때: "센서 정상" (한 줄)
이상일 때: 센서값 + 임계치 + 추이 (상세 블록)
```
- **사용**: 정상 상태에서는 정보량을 최소화
- **이상 감지 시 자동 확장**

## 컨텐츠 스펙 작성

새 화면이나 리디자인 시 아래 형식으로 스펙을 작성하라.

```markdown
## [화면명] 컨텐츠 스펙

### 운영자 시나리오
- Q1: [지금 정상인가?를 어떻게 답하는가]
- Q2: [어디가 문제인가?를 어떻게 답하는가]
- Q3: [왜/어떻게 대응하는가?를 어떻게 답하는가]

### Layer 0 (항상 노출)
| 항목 | 표현 형태 | 위치 |
|------|----------|------|

### Layer 1 (1클릭 노출)
| 항목 | 트리거 | 표현 형태 |
|------|--------|----------|

### Layer 2 (드릴다운)
| 항목 | 접근 방식 | 기본 상태 |
|------|----------|----------|

### 그룹핑
| 그룹명 | 포함 항목 | disclosure 패턴 |
|--------|----------|----------------|

### 제거/통합 대상
| 현재 | 변경 | 이유 |
|------|------|------|
```

## 산업용 대시보드 안티패턴

| 안티패턴 | 문제 | 대안 |
|---------|------|------|
| **정보 민주주의** | 모든 데이터를 동등하게 나열 | 우선순위 기반 계층화 |
| **테이블 만능** | 모든 데이터를 테이블로 표시 | 요약은 카드/뱃지, 상세만 테이블 |
| **패널 폭포** | 사이드 패널에 10개+ 섹션 flat 나열 | 탭 그룹 + 접기 |
| **중복 안전망** | "혹시 모르니 여기에도 표시" | 한 곳에만. 나머지는 링크 |
| **항상 상세** | 정상/이상 구분 없이 항상 상세 표시 | 정상 시 축소, 이상 시 확장 |
| **영어 혼용 제목** | "Calibration Metric", "Weight Log" | 사용자 언어로 통일 |

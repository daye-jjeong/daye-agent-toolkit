# 글쓰기 명확성 상세 가이드

## 능동태 우선

| 수동태 (지양) | 능동태 (권장) |
| --- | --- |
| "버그가 팀에 의해 발견되었습니다" | "팀이 버그를 발견했습니다" |
| "기능이 구현될 예정입니다" | "저희가 기능을 구현합니다" |
| "테스트 중 오류가 발견되었습니다" | "테스트에서 오류를 발견했습니다" |
| "서버가 재시작되었습니다" | "운영팀이 서버를 재시작했습니다" |
| "코드가 리뷰되어야 합니다" | "OO님이 코드를 리뷰해야 합니다" |

**원칙:** 주어를 명확히 하면 책임 소재도 명확해집니다.

## 불필요한 표현 제거

| 대신에 | 이렇게 |
| --- | --- |
| "현 시점에서" | "지금" |
| "~한 경우에" | "~하면" |
| "~라는 사실 때문에" | "~이므로" |
| "~하기 위해서" | "~하려고" |
| "혹시 확인 가능하실지 여쭤봅니다" | "확인 부탁드립니다" |
| "~에 대한 부분에 있어서" | "~에 대해" |
| "~하는 것이 필요한 상황입니다" | "~해야 합니다" |
| "말씀하신 내용과 관련하여" | "말씀하신 것처럼" |

**원칙:** 같은 뜻이면 짧은 표현이 항상 낫습니다.

## "그래서 뭐?" 테스트

작성한 메시지를 보내기 전, 이렇게 자문하세요:

> "그래서 뭐? 이게 읽는 사람에게 왜 중요한데?"

명확히 답할 수 없다면 메시지를 재구성해서 **가치/영향을 먼저** 제시하세요.

### 예시

**Before:**
> "캐시 레이어를 Redis에서 Memcached로 전환했습니다."

**After:**
> "응답 시간이 200ms → 50ms로 개선됐습니다. 캐시를 Redis에서 Memcached로 전환한 결과입니다."

**Before:**
> "PR #142를 머지했습니다."

**After:**
> "결제 페이지 로딩 속도 30% 개선이 반영됐습니다 (PR #142)."

## 이메일 구조 템플릿

```
제목: [프로젝트/주제]: [구체적 목적]

OO님 안녕하세요,

[핵심 요점이나 요청을 1-2문장으로 먼저]

**배경:**
- [요점 1]
- [요점 2]

**요청 사항:**
- [구체적 액션 또는 필요한 의사결정]
- [기한이 있다면 명시]

[선택: 간략한 다음 단계나 후속 계획]

감사합니다.
[이름]
```

```
- [결정 2]

**액션 아이템:**
- [ ] [담당자]: [할 일] - 기한: [날짜]
- [ ] [담당자]: [할 일] - 기한: [날짜]

**다음 단계:**
- [후속 미팅 필요 시]
- [공유할 문서]
```

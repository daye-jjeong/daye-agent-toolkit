# 업무 커뮤니케이션 가이드

소프트웨어 개발 환경에서 효과적인 업무 커뮤니케이션을 위한 프레임워크. 이메일, 팀 채팅, 미팅, 기술↔비기술 소통을 다룹니다.

**핵심 원칙:** 내가 얼마나 아는지 증명하는 게 아니라, 상대방이 정확히 이해하게 하는 것.

## 핵심 프레임워크

### 무엇-왜-어떻게 구조

모든 업무 메시지에 적용:

| 구성 | 목적 | 예시 |
| --- | --- | --- |
| **무엇** | 주제/요청을 명확히 | "릴리스를 1주 연기해야 합니다" |
| **왜** | 이유 설명 | "결제 처리에서 치명적 버그 발견" |
| **어떻게** | 다음 단계/액션 | "QA가 목요일까지 재테스트, 금요일 업데이트" |

### 글쓰기 3원칙

1. **명확한 제목/목적으로 시작** — 받는 사람이 즉시 파악
2. **글머리표와 소제목으로 스캔 가능하게** — 벽돌 텍스트 금지
3. **핵심 먼저** — 결론부터, 바쁜 사람은 효율을 고마워함

### 청중 맞춤

커뮤니케이션 전에 자문: (1) **누구에게** 쓰나? (2) **어느 정도의 디테일**이 필요한가? (3) **상대에게 어떤 가치**가 있나?

## 이메일

### 제목 공식

| 대신에 | 이렇게 |
| --- | --- |
| "프로젝트 업데이트" | "[프로젝트X] 주간 진행상황 및 다음 단계" |
| "질문" | "[질문] API 레이트 리미팅 방식 관련" |
| "참고" | "[FYI] 화요일 오후 3시 배포 예정" |

### 유형별 핵심

| 유형 | 핵심 요소 |
| --- | --- |
| **상태 보고** | 진행 요약, 차단 요소, 다음 단계, 일정 |
| **요청** | 명확한 요청, 맥락, 기한, 중요한 이유 |
| **에스컬레이션** | 이슈 요약, 영향, 시도한 해결책, 필요한 결정 |
| **공지/FYI** | 변경 내용, 영향 받는 사람, 필요 액션 |

**구조 템플릿 + 상세 가이드**: `{baseDir}/references/email-templates.md` 참고

## 팀 메시징

### 5대 원칙

1. **스레드 사용** — 메인 채널 깔끔하게
2. **@멘션은 신중하게** — 불필요한 알림 금물
3. **채널 구분** — 적합한 채널에 적합한 주제
4. **직접적으로** — "PR 리뷰 부탁드립니다" > "혹시 바쁘신가요?"
5. **비동기 친화적** — 즉각 응답 요구 금지

### 인사만 보내지 마세요

- ❌ "안녕하세요" → [대기] → "계세요?" → [대기] → 질문
- ✅ "안녕하세요 OO님 — 배포 스크립트 42번째 줄 권한 오류, 혹시 본 적 있으신가요?"

**비동기 커뮤니케이션 상세**: `{baseDir}/references/remote-async-communication.md` 참고

## 기술 vs 비기술 소통

| 청중 | 접근 |
| --- | --- |
| **개발 동료** | 기술 상세, 코드 예시, 아키텍처 |
| **기술 매니저** | 상세 + 고수준 영향 균형 |
| **비기술 이해관계자** | 비즈니스 영향, 비유, 결과 중심 |
| **고객** | 쉬운 말, 그들에게 어떤 의미인지 |

**단순화 전략**: 큰 그림부터 → 정확성 잃지 않으면서 단순화 → 상황 판단

**전문용어 번역**: `{baseDir}/references/jargon-simplification.md` 참고

## 글쓰기 명확성

- **능동태 우선** — "버그가 발견되었습니다" → "팀이 버그를 발견했습니다"
- **군더더기 제거** — "현 시점에서"→"지금", "~한 경우에"→"~하면"
- **"그래서 뭐?" 테스트** — 읽는 사람에게 왜 중요한지 답할 수 없으면 재구성

**상세 예시 + 템플릿**: `{baseDir}/references/writing-clarity.md` 참고

## 미팅

**미팅 전**: 명확한 목표 + 안건 + 사전 준비물 + 기대 결과를 초대에 포함
**미팅 중**: 시간 제한 설정, 실시간 액션 아이템 기록, 주차장(parking lot) 활용
**미팅 후**: 결정사항 + 액션 아이템(담당자/기한) + 다음 단계를 요약 공유

**미팅 유형별 구조 + 템플릿**: `{baseDir}/references/meeting-structures.md` 참고

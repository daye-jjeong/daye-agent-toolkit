# 마비노기 모바일 작곡 워크플로우

출처: [마비노기 MML연구소 (mabicompose.com)](https://www.mabicompose.com/)

---

## 모바일 제약

| 항목 | 제한 |
|------|------|
| 최대 트랙 | **6트랙 (6화음)** |
| 파트당 글자수 | **1200자** (보수적 기본값) |
| N명령 사용 여부 | 사용 가능 — 글자수 절약 효과 |

**글자수 제한 불확실성**: 소스마다 1200/2400으로 상이하며, 게임 업데이트로 변동 가능성이 있다. 스크립트는 안전한 기본값 1200을 사용한다. 실제 한계는 게임 내에서 직접 실측하여 확인할 것.

**PC판 악보 사용 불가**: 마비노기 모바일은 PC판과 악보 구조·연주 방식이 완전히 다르다. PC용 MML을 그대로 붙여넣으면 동작하지 않는다.

## 마비꼬(Mabiicco) export 절차

**요구 버전**: 마비꼬 1.5.7 이상

1. 마비꼬에서 편집 완료 후 **파일 > 내보내기** 선택
2. "N 명령 허용" 체크박스 **활성화** (글자수 절약)
3. 최대 **6트랙** 이내로 선택
4. **클립보드로 출력** 선택
5. 게임으로 전환 후 악보 편집 창에 **붙여넣기**

---

## MIDI 임포트

**양자화 정렬**: MIDI 임포트 시 **L64 (6tick)** 정렬 권장. 편집 오류 확률이 줄어든다.

**변환 리포트 손실 확인 필수**: MIDI 자동 변환은 "티가 많이 남"는다. 변환 후 반드시 리포트를 확인한다.

| 리포트 항목 | 의미 |
|------------|------|
| `skipped_chunks` | 음표 없는 청크로 건너뛴 청크 수 |
| `unmatched` | 매핑 실패 음표 수 |
| `quant_error` | 양자화 오차가 큰 음표 수 |
| `notes_dropped_polyphony` | 화음 정리로 제거된 음표 수 |
| `tracks_dropped` | 6트랙 초과로 잘린 트랙 수 |

값이 크면 마비꼬에서 수동 후처리가 필요하다. 주요 증상:

- 악기 증가 / 파트 섞임
- 박자 어긋남
- 글자수 초과

## 게임 내 단계

1. **빈 악보 구입** — 게임 내 상점 또는 NPC에서 구입
2. **가방에서 악보 클릭 > 편집** 선택
3. 직접 작곡 또는 **외부 MML 붙여넣기**
4. **"미리 듣기"** 버튼으로 재생 확인
5. **곡 제목 입력**
6. **"곡 만들기"** 버튼으로 최종 저장

---

## 박자 갭 수정

모바일은 박자 갭(음표 사이 공백)에 PC보다 민감하다.

**갭 제거 방법**: 마비꼬에서 해당 음표를 선택 → **Shift+Delete**

---

## "Error" · "Verify Error" 대응

게임 또는 마비꼬에서 `Error` / `Verify Error`가 표시되면:

1. 빨간 구간으로 이동
2. 주요 원인 확인:
   - 부적절한 위치의 템포(`t`) 명령
   - 음표 길이 오류 (0값, 비허용 값)
   - 글자수 초과
   - 트랙 수 6 초과
3. 해당 구간 수정 후 재확인

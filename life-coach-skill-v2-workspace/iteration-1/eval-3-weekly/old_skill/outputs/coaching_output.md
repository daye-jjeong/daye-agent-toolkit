# 주간 코칭 리포트 (2026-03-10 기준)

## 입력 데이터

```
총 세션: 18회
총 작업 시간: 32.5h
총 토큰: 1.2M
일별: 월 7h, 화 8h, 수 5h, 목 6h, 금 4.5h, 토 2h, 일 0h
태그: 코딩 40%, 디버깅 30%, 리서치 20%, 설계 10%
레포: daye-agent-toolkit 45%, cube-claude-skills 35%, life-dashboard 20%
운동: 2일 (30분씩), 식사: 14끼 (2끼 스킵), 평균 수면: 6.8h
반복 패턴: 새벽 1시 이후 작업 (3회)
```

---

### 📝 주간 정리

주 32.5시간, 18세션. 화요일(8h)이 피크, 토요일(2h) 이후 일요일은 완전 쉬었다. 레포별로는 daye-agent-toolkit(45%), cube-claude-skills(35%), life-dashboard(20%) 순으로 분산. 코딩(40%)과 디버깅(30%)이 전체의 70%를 차지했다.

### 🔍 방향성 코칭

디버깅이 30%로 높다 — 코딩 시간 대비 디버깅 비율이 크면 사전 설계나 테스트 커버리지를 점검할 필요가 있다. 새벽 1시 이후 작업이 3회 반복됐는데, 이번 주 후반(목·금)에 집중됐을 가능성이 높다. 작업 마감 시간을 정해두는 게 다음 주 수면 질 개선에 직접 영향을 준다.

### 🔮 다음 주 생각해볼 것

- 디버깅 30%의 원인이 설계 단계 부족인가, 아니면 외부 의존성 문제인가?
- 새벽 1시 이후 작업을 한 날 — 다음날 생산성이 어땠나?
- 운동 2일(30분씩)이 목표 대비 충분한가, 아니면 더 늘릴 여지가 있나?

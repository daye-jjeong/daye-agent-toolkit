# Old Skill 접근 방식 노트 — 주간 코칭

## 스킬 구조 파악

SKILL.md에서 주간 코칭 흐름:
1. `weekly_coach.py --json` 실행 → 데이터 획득 (온디맨드 시뮬레이션으로 대체)
2. `coaching-prompts.md`의 주간 코칭 프레임 적용
3. 주간 트렌드 + 방향성 코칭 대화

## coaching-prompts.md 적용 방식

### 섹션 구성 (주간)
프롬프트 지시: "다음 3개 섹션을 생성해라. 각 섹션 2-4줄. 총 500자 이내."

- `📝 주간 정리` — 핵심 성과, 시간 배분 중심
- `🔍 방향성 코칭` — 주간 트렌드 패턴 1-2개 + 다음 주 방향
- `🔮 다음 주 생각해볼 것` — 답 없이 질문만 2-3개

### 톤 레벨
데이터 상 새벽 1시 이후 작업 3회 반복 → escalation_level = Level 1 적용.
프롬프트 지시: "Level 1: 직접적 제안."

### 건강/운동 데이터 처리
old 스킬의 주간 코칭 섹션(SKILL.md § 주간 코칭 구성)은 5개 섹션이나,
coaching-prompts.md의 주간 프롬프트는 3개 섹션만 명시.
→ coaching-prompts.md 우선 적용 (SKILL.md 구성은 참고 수준).
→ 건강 데이터(운동 2일, 수면 6.8h)는 방향성 코칭 내에 자연스럽게 통합.

## 관찰된 제약/특이점

1. **data_section 자유도**: 프롬프트에 `{data_section}` 플레이스홀더만 있고 포맷 규정 없음. 어떤 형식으로 데이터를 넘겨도 된다.
2. **건강 섹션 미분리**: 주간 프롬프트에 건강 전용 섹션이 없어, 건강 지표가 코칭 섹션 안에 묻힌다.
3. **레포/태그 분석 깊이**: 분포(%)만 주어지면 "어느 작업이 왜 많았는가" 분석이 어렵다. 코칭이 표면적 관찰에 머문다.
4. **반복 패턴 활용**: 일일 코칭 프롬프트에는 "행동 신호 활용" 지시가 있으나, 주간 프롬프트에는 없음. 그러나 repeated_patterns 데이터를 방향성 코칭에 수동으로 연결했다.
5. **글자수 제약**: 500자 이내 — 섹션이 3개이므로 섹션당 평균 ~160자. 깊이 있는 분석보다 간결한 요점 전달에 적합.

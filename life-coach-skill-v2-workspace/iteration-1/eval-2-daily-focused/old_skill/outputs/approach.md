# Approach Notes — Old Skill (eval-2-daily-focused)

## 요청 vs 스킬 구조의 불일치

**사용자 요청:** "오늘 건강이랑 과작업 부분만 봐줘"
→ 특정 섹션만 필터링하는 focused/partial 출력 요청

**Old SKILL.md 구조:**
일일 코칭 워크플로우는 `coaching-prompts.md`의 고정 4섹션 템플릿을 그대로 생성한다:
1. 📝 오늘의 정리
2. 🔍 코칭
3. 🤖 자동화 제안
4. ⏭️ 내일 이어할 것

"건강만", "과작업만" 같은 focused 요청을 처리하는 명시적 메커니즘이 없다.

## Old Skill이 실제로 한 것

- 전체 템플릿 4섹션을 생성 (요청과 무관하게)
- "건강"과 "과작업" 내용은 코칭 섹션 안에 혼합되어 언급됨
- 자동화 제안, 내일 이어할 것 → 데이터 부족으로 생략 처리 (그나마 짧게)
- 건강 넛지가 별도 섹션 없이 코칭 섹션에 묻혀 있음

## 핵심 갭

| 항목 | Old Skill 동작 |
|------|----------------|
| 섹션 필터링 | 불가. 전체 템플릿 고정 출력 |
| 건강 전용 섹션 | 없음. 코칭 섹션 내 1-2줄로 처리 |
| 과작업 전용 분석 | 없음. 코칭 섹션 내 혼합 |
| 수면/수분/운동 개별 분석 | 없음. 통합 언급만 |
| 사용자 의도 파악 | 없음. 요청 텍스트 무시하고 고정 플로우 실행 |

## 결론

Old SKILL.md는 "전체 일일 리포트" 패턴으로 설계되어 있어서,
focused 요청("X 부분만 봐줘")을 처리하려면 스킬 구조 자체를 바꿔야 한다.
요청 파싱 → 섹션 선택 → 해당 섹션 심화 분석 흐름이 필요하다.

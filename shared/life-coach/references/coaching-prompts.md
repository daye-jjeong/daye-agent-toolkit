# 코칭 프롬프트 템플릿

## 일일 코칭

아래는 오늘 하루 작업 데이터이다.

{data_section}

다음 4개 섹션을 생성해라. 각 섹션 2-4줄. 총 600자 이내.

### 📝 오늘의 정리
- 하루 전체를 한 문단으로 요약.
- 성과 분류 기준:
  - **완료**: has_commits=1, 또는 summary에 "merge", "배포", "완료" 포함
  - **진행중**: 그 외 (설계중, 파일럿, 리서치 등)
- "완료"와 "진행중"을 명확히 구분해라. 진행중인 걸 완료된 것처럼 적지 마라.
- 캘린더 이벤트(source=calendar)가 있으면 미팅/운동 시간도 포함.

### 🔍 코칭
- 데이터에서 보이는 패턴 1-2개를 짚고, 구체적 제안.
- 패턴 예시:
  - 과작업(8h+), 새벽/주말 작업, 컨텍스트 스위칭, 운동 부재
  - 캘린더 미팅 + CC 작업 합산 시간
- **시간대 적절성**: 새벽/주말 작업이 있으면 해당 세션의 내용을 보고 "이 작업은 긴급했나? 다음 근무일로 미룰 수 있었나?" 판단. 단순히 "새벽 주의"가 아니라 구체적으로 어떤 작업이었는지 짚어라.
- 톤 레벨: {tone_level}
  - Level 0: 질문형 + 부드러운 넛지. "내일은 어떻게 할까?"
  - Level 1: 직접적 제안. "내일은 6시간 이하로 제한하는 게 좋겠다."
  - Level 2: 지시형. "내일 오후 7시 이후 작업 금지."

### 🤖 자동화 제안
- 반복되는 작업/명령이 있으면 자동화 방법 제안.
- 없으면 이 섹션 생략.

### ⏭️ 내일 이어할 것
- 진행중인 작업 중 다음에 이어야 할 것 1-2개 짚어라.
- 데이터에서 판단할 수 없으면 이 섹션 생략.

한국어로. 간결하게. 번역체 금지.

## 주간 코칭

아래는 이번 주 작업 데이터이다.

{data_section}

다음 3개 섹션을 생성해라. 각 섹션 2-4줄. 총 500자 이내.

### 📝 주간 정리
- 한 주 전체를 한 문단으로 요약. 핵심 성과와 시간 배분 중심.
- 성과 분류 기준은 일일 코칭과 동일 (완료 vs 진행중 구분).
- 캘린더 데이터가 있으면 미팅 시간 vs 코딩 시간 비율 포함.

### 🔍 방향성 코칭
- 주간 트렌드에서 보이는 패턴 1-2개를 짚고, 다음 주 방향 제안.
- 레포 집중도, 작업 유형 편중, 작업 시간 분포, 휴식 패턴 등.
- 톤 레벨: {tone_level}
  - Level 0: 질문형 + 부드러운 넛지.
  - Level 1: 직접적 제안.
  - Level 2: 지시형.

### 🔮 다음 주 생각해볼 것
- 데이터에서 관찰되는 패턴을 바탕으로 2-3개 질문.
- 답을 주지 말고 질문만 던져라.

한국어로. 간결하게. 번역체 금지.

# 프로파일 분석 차원

collect.py JSON 데이터를 해석하여 프로파일을 작성할 때 이 문서를 참조한다.

## 1. 시간 패턴

**데이터:** `sessions.by_weekday`, `sessions.by_hour`, `sessions.avg_duration_min`

**해석 기준:**
- 집중 시간대: `total_min` 기준 상위 3시간대
- 생산적 요일: `total_min` 기준 상위 요일
- 세션 길이 분포: 짧은 세션(<15분) 비율이 50% 이상이면 집중도 이슈
- 야간 작업(22시 이후) 비율

**강점 예시:** "심야 시간대에 긴 세션을 유지하며 깊은 집중력을 보인다"
**개선 예시:** "짧은 세션 비율이 높아 컨텍스트 스위칭이 잦다"

## 2. 작업 성향

**데이터:** `sessions.by_tag` (count + total_min 둘 다 확인)

**해석 기준:**
- 태그 비율은 `total_min` 기준으로 계산 (건수만 보면 짧은 세션이 과대 표현)
- 코딩 vs 설계 비율: 설계가 20% 이상이면 "설계 중심", 5% 미만이면 "실행 중심"
- 디버깅 비율: 25% 이상이면 주의 (근본 원인 분석 부족 가능성)
- 리팩토링 존재 여부: 코드 품질에 투자하는 성향 지표

**강점 예시:** "설계에 시간을 투자하여 구현 전 방향을 잡는 스타일"
**개선 예시:** "디버깅 비율이 높아 사전 테스트 강화가 필요할 수 있다"

## 3. 실수 패턴

**데이터:** `behavioral_signals.top_mistakes`, `behavioral_signals.repeat_signals`

**해석 기준:**
- 반복 신호 (`repeat_signals`)가 핵심 — 같은 내용이 3회+ 반복되면 구조적 문제
- 실수 유형을 카테고리화: 가정 오류, 도구 오용, 범위 초과, 테스트 누락 등
- 최근 실수와 과거 실수 비교 (개선 추세 여부)

**강점 예시:** "실수를 빠르게 인지하고 교정 규칙으로 체계화한다"
**개선 예시:** "'잘못된 가정' 유형이 반복 — 검증 단계를 추가할 필요"

## 4. 의사결정 패턴

**데이터:** `behavioral_signals.top_decisions`

**해석 기준:**
- 결정의 빈도: 많으면 자율적, 적으면 수동적
- 결정 내용에서 기술 선택, 아키텍처 판단, 우선순위 결정 비율
- "A 대신 B" 형태의 트레이드오프 결정이 있는지

**강점 예시:** "기술 선택에서 명확한 기준을 가지고 빠르게 결정한다"
**개선 예시:** "트레이드오프 결정 기록이 적어 판단 근거가 불분명"

## 5. 행동 패턴

**데이터:** `behavioral_signals.top_patterns`

**해석 기준:**
- 좋은 패턴과 나쁜 패턴을 분리
- 반복되는 좋은 패턴 → 강점 섹션에 반영
- 반복되는 나쁜 패턴 → 개선 포인트 후보

## 6. 도구/레포 선호

**데이터:** `sessions.by_repo`, `sessions.by_source`

**해석 기준:**
- source 비율 (CC vs Codex): 도구 선호도
- 레포 집중도: 상위 3개 레포가 전체의 80% 이상이면 "집중형"
- 멀티태스킹: 하루에 3+ 레포를 오가면 컨텍스트 스위칭 비용

**강점 예시:** "소수 프로젝트에 집중하여 깊이 있는 작업을 한다"
**개선 예시:** "매일 4-5개 레포를 오가며 컨텍스트 스위칭 비용이 높다"

## 7. 교정 이력

**데이터:** `corrections[]`

**해석 기준:**
- 교정 규칙의 카테고리 분포 (파일명 slug에서 추론)
- 규칙 생성 빈도: 주 2개 이상이면 빠른 학습자
- 규칙 내용에서 반복 주제 (예: worktree, testing이 반복되면 해당 영역 약점)

## 페르소나 작성 가이드

- **모든 서술은 데이터 근거를 명시**: "세션의 35%가 22시 이후"처럼 구체적 수치 포함
- **주관적 판단 최소화**: "성실하다", "게으르다" 같은 평가 금지. 관찰된 패턴만 기술
- **3인칭 서술**: "다예는 ~하는 경향이 있다"
- **200자 내외**: 핵심 특성 3-4가지만

## 개선 포인트 A/B 제시 포맷

각 개선 포인트에 대해:

```markdown
### [개선 포인트 제목]
- **근거**: 데이터에서 N회 관찰 (구체적 수치)
- **현재 방식**: 현재 어떻게 하고 있는지
- **제안 방식**: 대안 접근법
- **적용 방법**: 규칙 추가, 스킬 수정, 또는 행동 변화 제안
```

## 변화 추이 해석

이전 snapshot(`profile-snapshot.prev.json`)과 비교할 때:

- **유의미한 변화 기준**: ±10% 이상 또는 절대값 차이가 큰 경우
- **개선 확인**: 이전 프로파일의 "개선 포인트"가 데이터상 개선되었는지
- **새로운 패턴**: 이전에 없던 반복 신호나 행동 패턴
- **악화 감지**: 이전에 강점이었던 것이 약화된 경우
